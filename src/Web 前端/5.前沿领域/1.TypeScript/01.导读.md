`TypeScript`是一种强类型编程语言，它建立在 JavaScript 的基础上，能在任何规模上为您提供更好的工具。

TypeScript 是 JavaScript 的 超集 ：因此 JS 语法是合法的 TS。

## 安装

- 全局安装

```bash
npm i typescript -g
```

- 项目安装

```bash
npm i typesctipt -D
```

**检查**

```bash
tsc -v

# 输出版本号
# Version 5.8.3
```

---

**拓展**

其它编译 ts 的插件，如：`ts-node`

```bash
npm i ts-node -g
npm i @types/node -D
```

可以直接执行 ts 文件

```bash
ts-node demo.ts
```

## 编译

使用 tsc 命令将 ts 文件编译为 js 文件

```bash
tsc demo.ts

# 编译生成 demo.js 文件
```

自动监听文件变化，自动编译

```bash
tsc demo.ts -w
```

## 常用命令

- 编译文件

```bash
tsc xxx.ts
```

- 自动监听编译

```bash
tsc [xxx.ts] -W

# 不指定具体文件时，监听所有文件
```

- 生成 ts 配置文件 `tsconfig.json`

```bash
tsc --init
```

## 配置文件

### 初始化

```bash
tsc --init
```

- 检测编译

```bash
tsc -W
```

或者 vscode > 终端 > 运行任务(新建选项中) > tsc 监视

![tsc 监视](https://upyun-oss.mu00.cn/2025/09/30/1759202937380.png)

### 配置项

| 配置             | 说明                                                      |
| ---------------- | --------------------------------------------------------- |
| noImplicitAny    | 禁止使用隐含的 any 类型，如函数参数没有设置具体类型       |
| strictNullChecks | 开启时不否允许将 null、undefined 赋值给其他类型比如字符串 |
| target           | 转换成 JS 的版本                                          |
| strict           | 是否严格模式执行                                          |
| module           | 使用的模块系统                                            |
